## 🤖 AI Integration (v2)

AI в системе используется как дополнительный слой обогащения данных (enrichment layer) и не участвует в основном процессе поиска, фильтрации и ранжирования. Базовая логика сервиса остаётся полностью детерминированной: результаты формируются на основе данных Google Places, применяются заданные фильтры и выполняется прозрачное ранжирование. AI не влияет на состав или порядок выдачи, а лишь дополняет уже найденные объекты.
Основной сценарий применения — анализ пользовательских отзывов. Отзывы собираются и обрабатываются асинхронно, после чего AI генерирует краткое резюме (summary), выделяет ключевые плюсы и возможные недостатки, а также общий тон отзывов. Результаты сохраняются в базе данных и отображаются на странице дилера в виде компактного блока. Это позволяет пользователю быстрее оценить качество компании, не читая все отзывы, при этом не нарушая предсказуемость и прозрачность основного поиска.

### Roadmap

```
v1 → ручные фильтры (radius, rating, hours, types, contacts)
v2 → + AI summary — молча анализирует отзывы, показывает бриф под каждым дилером
```

AI не требует участия пользователя — никакого чата, никакого ввода. Работает автоматически в фоне, пользователь просто видит готовую выжимку.

---

### v2 — Как работает AI Summary

```
1. Дилер впервые попадает в БД (cache MISS)
         ↓
2. Place Details API → reviews[], rating, hours, contacts, types, priceLevel
         ↓
3. Весь контекст отправляется в Claude Haiku
         ↓
4. AI возвращает структурированный JSON
         ↓
5. Summary сохраняется в БД рядом с дилером
         ↓
6. Следующий запрос → из кэша, AI не вызывается
```

> AI вызывается **один раз на дилера**. Все последующие запросы — из кэша.

### Что анализирует AI

Haiku получает полный контекст из Places API:

```python
dealer_context = {
    "name":          place["displayName"],
    "rating":        place["rating"],
    "total_reviews": place["userRatingCount"],
    "price_level":   place.get("priceLevel"),
    "opening_hours": place.get("regularOpeningHours"),
    "open_now":      place.get("currentOpeningHours", {}).get("openNow"),
    "website":       place.get("websiteUri"),
    "phone":         place.get("nationalPhoneNumber"),
    "types":         place.get("types", []),
    "reviews":       [r["text"]["text"] for r in place.get("reviews", [])],
}
```

На основе этого AI определяет:

- позитивные паттерны (цены, скорость, честность)
- негативные паттерны (очереди, скрытые комиссии)
- упоминания иностранных языков (русский, арабский, английский)
- признаки работы с экспортом
- гарантия и ее соблюдение

### Стратегия загрузки — Eager

Резюме показывается сразу в списке, без дополнительного клика.

➕ лучший UX — ценность продукта видна сразу  
➖ платишь за дилеров которых не открыли

**Решение — умный кэш:** первый пользователь который запросил дилера платит $0.004. Все остальные — из кэша бесплатно. На практике при 100 DAU в одном городе база быстро прогревается.

### Ограничения

- Google Places API отдаёт максимум **5 отзывов** на место
- `export_friendly` — эвристика на основе отзывов, не верифицированный факт
- если отзывов нет — AI-блок не отображается

### ⚠️ AI Disclaimer(AGB)

AI-резюме формируется автоматически на основе отзывов из Google Places и носит **информационный характер**.

Сервис не гарантирует точность, полноту или актуальность сгенерированных данных. Все выводы AI являются эвристическими и могут не отражать реальное положение дел. Пользователь самостоятельно принимает решение на основе предоставленной информации.
